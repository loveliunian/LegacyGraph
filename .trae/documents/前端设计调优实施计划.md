## 一、现状分析 (Current State Analysis)

### 1.1 已探索的关键架构

**前端技术栈**:
- Vue 3.4 + TypeScript
- Element Plus 2.14 (UI 组件库)
- Pinia (状态管理，支持持久化)
- Vue Router (路由)
- ECharts 5.5 (图表)
- Vue Flow 1.33 (图谱可视化)
- Vite 5.1 (构建工具)

**关键入口文件**:
- [frontend/index.html](file:///Users/huymac/工作/数智/LegacyGraph/frontend/index.html) - HTML 入口，引入字体位置
- [frontend/src/main.ts](file:///Users/huymac/工作/数智/LegacyGraph/frontend/src/main.ts) - 应用入口，初始化主题
- [frontend/src/styles/variables.css](file:///Users/huymac/工作/数智/LegacyGraph/frontend/src/styles/variables.css) - CSS 变量定义

**主题系统架构**:
- [frontend/src/stores/app.ts](file:///Users/huymac/工作/数智/LegacyGraph/frontend/src/stores/app.ts) - 应用状态管理
  - 支持 light/dark/auto 三种主题模式
  - `isDark` computed 属性判断当前主题
  - `toggleTheme()` 方法切换主题
  - 支持动态更新主题配置 (`updateThemeConfig`)
  - 已有主题持久化机制 (localStorage)

**当前字体系统**:
- 使用系统默认字体栈: `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, Oxygen, Ubuntu, Cantarell, 'Fira Sans', 'Droid Sans', 'Helvetica Neue'`
- 缺少品牌字体，视觉记忆点不足
- 标题与正文缺少层次区分

**当前配色系统**:
- Element Plus 默认主色: `#409eff` (常见蓝色)
- 已支持暗色模式但配色较为保守
- 已在 [variables.css](file:///Users/huymac/工作/数智/LegacyGraph/frontend/src/styles/variables.css) 定义 CSS 变量

## 三、假设与决策 (Assumptions & Decisions)

### 3.1 设计决策

**主色选择**: #6366F1 (Indigo-500)
- **理由**: 区别于常见的蓝色 (#409eff)，体现独特性；Indigo/Violet 色系体现智能、科技、数据科学气质
- **暗色模式**: #818CF8 (Indigo-400) - 更亮的 Indigo，在深色背景下更突出

**字体选择**: DM Serif Display + DM Sans + JetBrains Mono
- **Display Font**: DM Serif Display - 几何感强、现代优雅、具有数据科学的专业气质
- **UI Font**: DM Sans - 高可读性、现代精致、与 Display Font 协调
- **Code/Data Font**: JetBrains Mono - 等宽、清晰、专为代码和数据设计

**动效策略**: 编排化加载 + 微交互 + 图谱动画
- **页面加载**: 关键元素依次入场 (0.1s-0.6s 延迟)
- **微交互**: 按钮 hover 上浮、卡片 hover 上浮、图谱节点放大
- **图谱动画**: 节点脉冲效果、边流动动画（可选）

**布局策略**: Grid 不对称布局
- **Dashboard**: 左侧数据区占主要空间 (1fr)，右侧面板固定宽度 (280px)
- **LoginPage**: 居中浮动卡片 + 半透明背景 + 模糊效果
- **Graph**: 图谱优先，工具栏浮动在图谱上

### 3.2 技术决策

**CSS 变量系统**: 所有设计元素使用 CSS 变量
- **优点**: 便于维护、支持暗色模式切换、系统化管理
- **实施**: 在 `variables.css` 中定义，全局引用

**Element Plus 兼容**: 替换 Element Plus 主色变量
- **风险**: 可能影响部分组件样式
- **缓解**: 全面测试所有 Element Plus 组件在新配色下的表现

**动画性能**: 使用 `cubic-bezier` 和 `will-change`
- **优化**: 避免过多同时运行的动画
- **监控**: 使用 Chrome DevTools Performance 工具

### 3.3 风险决策

**激进实施策略**: 不提供"经典模式"备用方案
- **理由**: 快速建立品牌识别度，避免设计碎片化
- **风险**: 用户接受度风险
- **缓解**: 分阶段实施，收集用户反馈

**字体依赖 Google Fonts CDN**
- **风险**: 网络依赖，可能影响加载速度
- **缓解**: 使用 `preconnect` 预连接，字体文件会被浏览器缓存
