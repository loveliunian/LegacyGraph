## P0-1：修复证据链断裂

### 问题现状

**证据在 Claim 转换时被丢弃**：`DocUnderstandingAgent.toClaimDrafts()` 将 LLM 返回的 `evidenceText` / `EvidenceRef` 全部丢弃，`evidenceIds` 写入的是文件路径而非真实 Evidence ID。

- [DocUnderstandingAgent.java#L239-L255](file:///Users/huymac/工作/数智/LegacyGraph/backend/src/main/java/io/github/legacygraph/agent/DocUnderstandingAgent.java#L239-L255)：`draft()` 方法中 `.evidenceIds(hasText(sourcePath) ? List.of(sourcePath) : List.of())` 直接把 `sourcePath` 当证据 ID
- LLM 返回的 `BusinessProcess.evidence`（`List<EvidenceRef>`）、`BusinessDomain.evidenceText`、`Evidence.contentExcerpt` 均未被持久化

**节点已存在时不追加证据**：`EvidenceGraphWriter.upsertNode()` 仅在 `upsert.created()` 为 true 时创建证据，命中已存在节点则跳过，导致同一节点的多来源证据丢失。

- [EvidenceGraphWriter.java#L144](file:///Users/huymac/工作/数智/LegacyGraph/backend/src/main/java/io/github/legacygraph/builder/EvidenceGraphWriter.java#L144)：`if (upsert.created() && ...)` 条件排除了已存在节点的证据追加

#### 2.2 修复 mergeByKey 置信度取值

```java
private static <T> List<T> mergeByKey(...) {
    Map<String, T> seen = new LinkedHashMap<>();
    for (var r : results) {
        for (T item : safe(getter.apply(r))) {
            String key = keyFn.apply(item);
            T existing = seen.get(key);
            if (existing == null) {
                seen.put(key, item);
            } else {
                // 取较高置信度（通过反射获取 confidence 字段）
                double existingConf = getConfidence(existing);
                double newConf = getConfidence(item);
                if (newConf > existingConf) {
                    seen.put(key, item);
                }
                // 合并 evidence 列表（多来源证据累加）
                mergeEvidence(existing, item, seen, key);
            }
        }
    }
    return new ArrayList<>(seen.values());
}
```

```java
public class ReportTruthPolicy {

    public static Predicate<KnowledgeClaim> forBody() {
        return c -> "CONFIRMED".equals(c.getStatus());
    }
    
    // 附录含 PENDING_CONFIRM / INFERRED，明确标识
    public static Predicate<KnowledgeClaim> forAppendix() {

                || "INFERRED".equals(c.getStatus());

    // REJECTED / STALE / CONFLICTED 不得进入结论
    public static Predicate<KnowledgeClaim> excluded() {
        return c -> "REJECTED".equals(c.getStatus()) 
                || "STALE".equals(c.getStatus())
                || "CONFLICTED".equals(c.getStatus());
    }
}
```

## P0-5：重做"准确率"指标

### 问题现状

**当前 `assessAccuracy()` 只检查端点非空**：[GraphQualityAssessor.java#L149-L170](file:///Users/huymac/工作/数智/LegacyGraph/backend/src/main/java/io/github/legacygraph/service/scan/GraphQualityAssessor.java#L149-L170)，抽样边的 `fromNodeId` / `toNodeId` 非空即算"准确"，空图谱得 100%。这属于引用完整性，不是语义准确率。

### 修复方案

将"准确率"拆为 6 个独立指标：

#### 5.1 结构完整性

```java
private StructuralIntegrityMetric assessStructuralIntegrity(String projectId, String versionId) {
    long danglingEdges = graphDao.countDanglingEdges(projectId, versionId);  // 端点不存在的边
    long duplicateNodes = graphDao.countDuplicateNodes(projectId, versionId); // 同 nodeKey 多节点
    long constraintViolations = assessConstraints(projectId, versionId).stream()
            .mapToLong(ConstraintViolation::getViolationCount).sum();
    return new StructuralIntegrityMetric(danglingEdges, duplicateNodes, constraintViolations);
}
```

需在 `Neo4jGraphDao` 中新增：
- `countDanglingEdges()`：查询 from/to 节点不存在的边
- `countDuplicateNodes()`：按 `(projectId, versionId, nodeType, nodeKey)` 分组计数 >1 的

#### 5.2 三元组准确率

抽样 N 条边，逐条检查是否由源资料支持（有关联 Evidence 且 Evidence 内容能支撑该关系）：

```java
private TripleAccuracyMetric assessTripleAccuracy(String projectId, String versionId) {
    List<GraphEdge> sampleEdges = graphDao.sampleEdgesWithEvidence(projectId, versionId, 100);
    int supported = 0;
    for (GraphEdge edge : sampleEdges) {
        List<Evidence> evidence = evidenceRepository.findByEdgeId(edge.getId());
        if (!evidence.isEmpty() && hasSupportingContent(edge, evidence)) {
            supported++;
        }
    }
    return new TripleAccuracyMetric(sampleEdges.size(), supported, 
            sampleEdges.isEmpty() ? 0 : (double) supported / sampleEdges.size());
}
```

#### 5.3 证据覆盖率

```java
private EvidenceCoverageMetric assessEvidenceCoverage(String projectId, String versionId) {
    long totalNodes = graphDao.countNodes(projectId, versionId, null);
    long nodesWithEvidence = nodeEvidenceRepository.countDistinctNodeIds(projectId, versionId);
    long totalEdges = graphDao.countEdges(projectId, versionId, null);
    long edgesWithEvidence = edgeEvidenceRepository.countDistinctEdgeIds(projectId, versionId);
    return new EvidenceCoverageMetric(
            totalNodes, nodesWithEvidence, (double) nodesWithEvidence / Math.max(1, totalNodes),
            totalEdges, edgesWithEvidence, (double) edgesWithEvidence / Math.max(1, totalEdges));
}
```
