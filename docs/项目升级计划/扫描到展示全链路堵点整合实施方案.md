## 2. 设计原则

| 原则 | 说明 |
|---|---|
| 复用优先 | 现有 `GraphBuilder` / `JavaMemberCallResolver` / `HybridRetrievalService` / `EvidenceVerifier` 优先扩展 |
| 版本边界 | `GraphRelease.status = PUBLISHED` 作为可读前提；首期允许 Neo4j 直读，便于回滚 |
| 权限分层 | 所有新增实体同时携带 `projectId` + `aclHash` 或 `principals` |
| Job/HTTP 分离 | 耗时任务走 `task/` + `service/`，API 只做编排 |
| 可视可调试 | 每条修复附"UI 复现 → 修复后表现" + 验收用例 |
| TDD 强制 | 每个任务至少 1 个单元测试 + 1 个集成回归 |

## 3. 实施路线图（4 个 Sprint）

| Sprint | 时长 | 主题 | 涉及堵点 | 退出标准 |
|---|---|---|---|---|
| S1 | 1 周 | 源码单轨 + 扫描锁 | H-D01 / H-D02 / H-S01 / H-S02 / H-D03 + M-D01 / M-D02 | `source_snapshot` 成为唯一来源，`file_snapshot` 进入只读归档；同 project 并发扫描无互踩 |
| S2 | 2 周 | 抽取层增强 | H-E01 / H-E02 / H-E03 / H-E04 + M-E01 / M-E02 / L-E01 | CALLS 边补全 ≥95%，8 类关键边全部连通，前端可点击跳源码 |
| S3 | 1 周 | 建图幂等 + 查询护栏 | H-G01 / H-G02 / H-G03 / H-Q01 + M-G01 / M-G02 / M-Q01 / M-Q02 / L-Q01 | Neo4j 无重复节点；路径查询 P99 < 2s；ACL 过滤有性能采样 |

## 4. Sprint 1：源码单轨 + 扫描锁（1 周）

### 4.1 目标
- 消除 `file_snapshot` / `source_snapshot` 双轨制
- 引入 `scan_lock` 防止并发扫描互踩
- `parse_failure` 暴露到前端

### 4.2 任务清单

| ID | 任务 | 涉及文件 | 验收 |
|---|---|---|---|
| S1-T1 | `SourceSnapshotService` 切换为唯一落库入口；`file_snapshot` 表进入只读视图 | `service/source/SourceSnapshotService.java`、`service/scan/FileChangeDetector.java` | 同文件二次扫描不再产生重复行；归档视图可读不可写 |
| S1-T2 | 引入 PG advisory lock 实现 `scan_lock(project_id)` | `task/ProjectScanner.java`、`entity/ScanVersion.java` | 并发触发扫描，第二个任务直接拒绝并提示"已有扫描在进行" |
| S1-T3 | `ScanCheckpoint` 提升为扫描主流程状态机唯一来源 | 新增 `entity/ScanCheckpoint.java`、`repository/ScanCheckpointRepository.java` | 断点续扫可恢复，未消费字段纳入主流程 |
| S1-T4 | `ParseFailure` 暴露到 `SourceController`，新增失败列表 + 重试入口 | `controller/SourceController.java`、新增 `ParseFailureRepository.java` | 前端可看到失败原因，可点重试 |
| S1-T5 | `DbConnection` 读路径切到 `SecretCipher` | `entity/DbConnection.java`、`service/security/LocalAesGcmSecretCipher.java`、`service/security/SecretCipher.java` | 读出的密码解密成功；V79 旧明文一次性回填脚本执行 |

### 4.3 风险与回滚
- 双轨制收口保留 `file_snapshot` 只读窗口 7 天
- 迁移脚本配套：`V__file_snapshot_archive_view.sql`（回滚只需 `DROP VIEW`）

## 5. Sprint 2：抽取层增强（2 周）

### 5.1 目标
- 静态分析覆盖 lambda / 方法引用 / 注解驱动 Bean / 远程调用 / 并发原语
- 8 类关键边全部建通
- 抽取结果携带源码定位 span，前端可跳转

### 5.2 任务清单

| ID | 任务 | 涉及文件 | 验收 |
|---|---|---|---|
| S2-T1 | `JavaMemberCallResolver` 引入方法句柄符号表 + lambda 捕获变量追踪 | `builder/JavaMemberCallResolver.java` | CALLS 边补全从 30~70% 提到 ≥95% |
| S2-T2 | `JavaStructureExtractor` 维护注解→节点类型映射表 | `extractors/JavaStructureExtractor.java` | `@Service` / `@Repository` / `@Mapper` 节点入库 |
| S2-T3 | `ServiceCallExtractor` 接入 `ConcurrencyPropertyExtractor` 的方法调用图 | `extractors/ServiceCallExtractor.java`、`concurrency/ConcurrencyPropertyExtractor.java` | Feign / Dubbo / HTTP 调用边入库 |
| S2-T4 | 并发原语抽取结果写回 Neo4j `Concurrency` 节点 + 边 | `concurrency/ConcurrencyPropertyExtractor.java`、`dao/Neo4jGraphDao.java` | `synchronized` / `Lock` / `CompletableFuture` 节点可查 |
| S2-T5 | `DocumentParsingRouter` 策略分发优先级外置配置 + 超时回退链 | `service/parse/DocumentParsingRouter.java`、`FastParsingStrategy.java`、`LayoutParsingStrategy.java`、`OcrParsingStrategy.java` | OCR/Layout/Fast 三策略超时自动回退 |
| S2-T6 | 抽取结果携带 `startLine` / `endLine` / `column` 定位 span | 抽取器统一接口 | 前端 `locateNode` 可跳转到源码对应行 |

### 5.3 风险与回滚
- 注解映射表采用"白名单 + 兜底日志"，未知注解不破坏既有流程
- OCR 失败兜底走 Fast 解析

## 6. Sprint 3：建图幂等 + 查询护栏（1 周）

### 6.1 目标
- Neo4j 无重复节点
- 路径查询带深度上限 + 熔断
- ACL 过滤有性能采样

### 6.2 任务清单

| ID | 任务 | 涉及文件 | 验收 |
|---|---|---|---|
| S3-T1 | `Neo4jGraphDao` 引入 `scan_version + node_key` 复合唯一约束，先 MERGE 再 UPSERT | `dao/Neo4jGraphDao.java`、`common/NodeType.java` | 重跑扫描无重复节点 |
| S3-T2 | `PgEvidenceTxExecutor` 拆分为"建图小事务 + 异步对账"两阶段 | `builder/PgEvidenceTxExecutor.java` | 长事务消失，PG 锁竞争下降 |
| S3-T3 | `Neo4jSchemaRepository` 启动期只补约束，索引创建走后台 job | `dao/Neo4jSchemaRepository.java` | 启动时间不随数据量线性增长 |
| S3-T4 | `GraphPathReadModel` 加 depth 软上限（默认 8）+ 超时熔断 | `service/graph/GraphPathReadModel.java`、`service/graph/GraphQueryService.java` | P99 < 2s，超时返回友好降级 |
| S3-T5 | `GraphQueryService` 缓存 key 加 `scan_version` 隔离 | `service/graph/GraphQueryService.java` | 不同版本查询不串数据 |
| S3-T6 | `ImpactAnalysisService` 影响面分层（直接/传递） | `service/graph/ImpactAnalysisService.java` | 返回结构含 `direct` / `transitive` 字段 |
| S3-T7 | `Neo4jProjectionRepository` 中文字段名加索引 + `Neo4jQueryRepository` prepared statement 缓存 | `dao/Neo4jProjectionRepository.java`、`dao/Neo4jQueryRepository.java` | 中文字段查询 P99 < 500ms |
| S3-T8 | `AclFilterService` 性能采样埋点 | `service/acl/AclFilterService.java`、`service/acl/AccessContext.java` | Micrometer 指标可见 |

### 6.3 风险与回滚
- 约束新增配套 `V__neo4j_orphan_cleanup.sql` 清理脚本
- 缓存命中失败时直接走 DB，不阻塞主链路
