# QA 问答页与图谱展示设计方案

> **设计原则**: Linear-style 克制美学 - 平静的表面层次、强烈的排版和间距、少量颜色、最小的 chrome

---

## 一、QA 问答页设计方案

### 1.1 视觉主题

**视觉 thesis**: 现代对话界面，平静而有层次的表面设计，以内容为主，减少装饰性元素

**核心设计理念**:
- 去掉卡片包装，使用简单的布局和间距
- 用户消息右对齐，助手消息左对齐，视觉区分明显
- 证据展示使用精致的芯片设计，悬停显示详情
- 输入区域简洁精致，与整体风格统一

### 1.2 布局重构

**整体布局**:
```
┌─────────────────────────────────────────────────────────────┐
│  [新对话] ────────────────────────────────────────── [折叠] │  ← 顶部工具栏
├─────────────────────────────────────────────────────────────┤
│  对话列表区域 ┆                    聊天内容区域               │
│  ┌─────────┐ ┆  ┌──────────────────────────────────────┐   │
│  │ 对话1   │ ┆  │ 用户消息 (右对齐)                      │   │
│  │ 对话2   │ ┆  │                                      │   │
│  │ 对话3   │ ┆  │ 助手消息 (左对齐)                      │   │
│  └─────────┘ ┆  │  • 证据芯片                           │   │
│              ┆  │  • 置信度                             │   │
│              ┆  │                                      │   │
│              ┆  │ 输入区域                              │   │
│              ┆  └──────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

**关键设计变更**:
- 去掉外层 `el-card`，使用纯布局
- 对话列表区域使用紧凑设计，hover 高亮
- 聊天区域使用全屏宽度，消息自然流
- 输入区域独立于消息区域，固定在底部

### 1.3 消息样式设计

**用户消息**:
- 背景: `var(--el-color-primary)` 主色
- 文字: 白色
- 圆角: 16px (左上和左下直角)
- 对齐: 右对齐
- 动画: fadeInUp

**助手消息**:
- 背景: `var(--el-fill-color)` 填充色
- 文字: `var(--el-text-color-primary)`
- 圆角: 16px (右上和右下直角)
- 对齐: 左对齐
- 动画: fadeInUp，延迟 0.1s

**消息内容**:
- 使用 Markdown 渲染
- 代码块使用 `var(--font-data)` 字体
- 列表使用主题色圆点

### 1.4 证据展示设计

**证据芯片**:
- 圆形设计，显示数字编号
- 颜色区分来源类型:
  - 图谱节点: 主色 `var(--el-color-primary)`
  - 文档: Teal `#14B8A6`
- hover 效果: 放大 + 阴影
- 点击弹出下拉菜单

**证据详情预览**:
- hover 时显示 tooltip，包含标题、来源文件、摘要
- 使用 `el-popover` 组件

### 1.5 输入区域设计

**输入框**:
- 无边框设计，使用底部边框
- 背景: `var(--el-bg-color)`
- 聚焦时底部边框变为主题色
- 占位符文字: `var(--el-text-color-placeholder)`

**操作按钮**:
- 发送按钮: 主色背景，圆角设计
- 清空按钮: 文字按钮
- 字符计数: 灰色小字

### 1.6 对话列表设计

**对话项**:
- 标题: `var(--font-heading)` 字体
- 元数据: `var(--font-size-caption)` 灰色
- 选中状态: 左侧主题色竖条
- hover 效果: 背景色变化

**新增对话按钮**:
- 固定在顶部
- 使用主题色背景

## 二、统一知识图谱设计方案

### 2.1 视觉主题

**视觉 thesis**: 数据可视化美学，清晰的层次结构，图谱优先，辅助信息精简

**核心设计理念**:
- 图谱区域全屏展示，去掉卡片包装
- 顶部工具栏简洁，融入统计信息
- 右侧面板固定宽度，可折叠
- 节点详情使用侧边滑出面板

### 2.2 布局重构

**整体布局**:
```
┌─────────────────────────────────────────────────────────────┐
│  统一知识图谱         [版本选择] [刷新]          节点: 1234  │
│                                                   关系: 5678 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                    图谱可视化区域                            │
│                   (全屏展示)                                │
│                                                             │
│                                                             │
│                                                             │
│                    ┆                                        │
│                    ┆  右侧面板 (固定 280px)                 │
│                    ┆  ┌────────────────────────────────┐   │
│                    ┆  │ 筛选器 (可折叠)               │   │
│                    ┆  │ • 节点类型                    │   │
│                    ┆  │ • 置信度范围                  │   │
│                    ┆  │ • 数据来源                    │   │
│                    ┆  │ • 审核状态                    │   │
│                    ┆  └────────────────────────────────┘   │
│                    ┆  ┌────────────────────────────────┐   │
│                    ┆  │ 节点详情 (选中时显示)          │   │
│                    ┆  │ • 节点名称                    │   │
│                    ┆  │ • 置信度进度条                │   │
│                    ┆  │ • 审核状态                    │   │
│                    ┆  │ • 快捷操作                    │   │
│                    ┆  └────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**关键设计变更**:
- 去掉 `el-card` 包装，图谱区域直接展示
- 顶部统计条融入工具栏，使用简洁的数字展示
- 右侧面板固定宽度 280px，使用 sticky 定位
- 筛选器可折叠，节省空间

### 2.3 顶部工具栏设计

**布局**:
- 左侧: 页面标题 + 描述
- 中间: 版本选择 + 刷新按钮
- 右侧: 统计数据 (节点数、关系数、待审核数、平均置信度)

**统计数据**:
- 使用紧凑的数字展示
- 数字使用 `var(--font-display)` 字体
- 标签使用 `var(--font-size-caption)` 灰色

### 2.4 图谱区域设计

**背景**:
- 使用 `var(--el-bg-color-page)` 背景色
- 深色模式下使用深色背景
- 添加细微的网格纹理 (可选)

**工具栏**:
- 图谱上方的工具栏固定
- 包含: 缩放、平移、布局切换、导出等操作
- 使用图标按钮，hover 显示 tooltip

### 2.5 右侧面板设计

**筛选器**:
- 可折叠设计，点击标题展开/收起
- 使用 `el-collapse` 组件
- 节点类型使用颜色圆点 + 复选框
- 置信度使用滑块
- 数据来源和审核状态使用复选框

**节点详情**:
- 选中节点时显示
- 顶部: 节点图标 + 名称 + 类型标签
- 中间: 置信度进度条 + 审核状态 + 证据数量
- 底部: 快捷操作按钮 (确认、拒绝、查看证据)

### 2.6 节点详情滑出面板 (可选)

**设计方向**:
- 使用右侧滑出面板替代固定面板
- 点击节点时从右侧滑出
- 宽度: 360px
- 包含完整的节点信息和操作
- 关闭时滑出面板收回

### 3.1 视觉主题

**视觉 thesis**: 功能模块导航 + 流程可视化 + 详情展示，三区域清晰区分

**核心设计理念**:
- 左侧模块列表使用紧凑设计
- 中间图谱区域全屏展示
- 右侧详情使用简洁布局，去掉卡片包装

### 3.2 布局重构

**整体布局**:
```

│  功能图谱                              [导出功能清单]      │
├─────────────────────────────────────────────────────────────┤
│  模块列表 ┆                图谱区域                 ┆ 详情 │
│  ┌──────┐ ┆  ┌─────────────────────────────────────┐ ┆    │
│  │模块1 │ ┆  │ 功能流程图谱                        │ ┆    │

│  └──────┘ ┆  │                                    │ ┆    │
│          ┆  │ 节点数: X / 关系数: X                │ ┆    │
│          ┆  └─────────────────────────────────────┘ ┆    │
│          ┆                                          ┆    │
│          ┆                                          ┆    │
│          ┆                                          ┆    │
│          ┆                                          ┆    │
└─────────────────────────────────────────────────────────────┘
```

**关键设计变更**:
- 左侧模块列表去掉卡片包装，使用简单边框
- 中间图谱区域去掉卡片包装，全屏展示
- 右侧详情区域使用简洁布局，去掉卡片包装

### 3.3 模块列表设计

**模块项**:
- 左侧: 彩色图标背景
- 右侧: 模块名称 + 功能点数量
- 选中状态: 左侧主题色竖条
- hover 效果: 背景色变化

### 3.4 图谱区域设计

**图谱工具栏**:
- 顶部: 当前模块名称 + 节点/关系统计
- 图谱: 功能流程图谱，使用 Vue Flow

### 3.5 详情区域设计

**空状态**:
- 显示提示文字，引导用户点击节点

**节点详情**:
- 使用简洁的布局，去掉 `el-descriptions` 表格
- 使用 `div` 布局，标签 + 值
- 标签使用灰色小字，值使用主色

## 八、关键设计原则回顾

### Linear-style 克制美学
- ✅ 平静的表面层次
- ✅ 强烈的排版和间距
- ✅ 少量颜色
- ✅ 最小的 chrome
- ✅ 卡片只在需要交互时使用

### 避免的设计
- ❌ 卡片堆叠布局
- ❌ 厚边框
- ❌ 装饰性渐变
- ❌ 多个竞争的强调色
- ❌ 装饰性图标

### Litmus Checks
- ✅ 品牌在第一屏幕是否清晰？
- ✅ 是否有一个强视觉锚点？
- ✅ 仅扫描标题能否理解页面？
- ✅ 每个区域是否只有一个职责？
- ✅ 卡片是否必要？
- ✅ 动效是否改善了层次或氛围？

## 九、总结

本设计方案旨在将 QA 问答页和图谱展示页面从传统的卡片堆叠布局升级为 Linear-style 克制美学，通过去掉冗余的卡片包装、优化视觉层次、增强动效体验，打造现代、专业的数据可视化平台。

**核心改进**:
1. QA 问答页: 消息气泡设计、证据芯片优化、输入区域精简
2. 统一知识图谱: 图谱全屏展示、工具栏精简、右侧面板优化
3. 功能图谱: 三栏布局优化、模块列表精简、详情区域优化

**预期效果**:
- 更清晰的视觉层次
- 更好的用户体验
- 更现代的设计感
- 更强的品牌识别度
