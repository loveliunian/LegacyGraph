### H05 runCommandGate RCE 修复（最高优先级安全）

**对应体检发现**：§1.5 异常吞咽（生产关键路径）

**现状**：
- `service/test/ValidationGateRunner.java:140-155` 的 `runCommandGate` 用 `/bin/sh -c {command}` 执行 `gate.getCommand()`
- `lg_validation_gate` 表被攻陷 → 任意命令执行 RCE

**修改计划**：

1. **新增配置类 `service/test/GateCommandWhitelist.java`**：
   ```java
   private static final Set<String> ALLOWED_BINARIES = Set.of(
       "mvn", "gradle", "npm", "node", "git", "make", "pytest", "go", "java", "python3"
   );
   ```
2. **`ValidationGateRunner.runCommandGate`** 改写：
   ```java
   private Process startGateProcess(ValidationGate gate, File workDir) {
       List<String> cmd = parseAndValidate(gate.getCommand());  // 切 tokens + 白名单校验
       String binary = cmd.get(0);
       if (!ALLOWED_BINARIES.contains(binary)) {
           throw new BizException("GATE_BINARY_NOT_WHITELISTED",
               "Binary '" + binary + "' is not in the allowed list");
       }
       ProcessBuilder pb = new ProcessBuilder(cmd).directory(workDir);
       return pb.start();  // 不再用 /bin/sh -c
   }
   ```
3. **新增 admin 鉴权**：在 `@PostMapping` 调用 `runGate` 前鉴权 `role=ADMIN`
4. **审计写入**：`lg_audit_log` 记录 `gateId / operator / command / exitCode / durationMs`
5. **回归测试**：
   - 用例 1：合法命令 `mvn test` → 200
   - 用例 2：非法命令 `rm -rf /` → 403 GATE_BINARY_NOT_WHITELISTED
   - 用例 3：嵌入命令 `mvn test; rm -rf /` → 切 token 后第二个被拒绝

**验收标准**：
- `mvn test -Dtest='ValidationGateRunnerTest,GateCommandWhitelistTest'` 通过
- 安全组复测：手动 `curl -X POST .../run-all` 提交 `command="rm -rf /"` 返回 403
