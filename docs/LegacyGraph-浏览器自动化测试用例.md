## 一、测试环境配置

### 1.1 环境准备

| 配置项 | 说明 |
|--------|------|
| 前端地址 | http://localhost:5173 |
| 后端地址 | http://localhost:8080/api |
| 浏览器 | Chrome / Chromium（最新版） |
| 视口大小 | 1920 x 1080（推荐桌面端） |
| 网络状态 | 正常网络（非弱网） |

### 1.2 前置条件

1. 后端服务已启动并正常运行
2. 前端开发服务已启动
3. 数据库（PostgreSQL + Neo4j + Redis）已就绪
4. 测试账号已创建
5. 至少存在一个测试项目，且已完成至少一次扫描

### 1.3 agent-browser 使用说明

- 使用 `navigate` 指令访问页面
- 使用 `click` 指令点击元素
- 使用 `type` 指令输入文本
- 使用 `snapshot` 获取页面结构
- 使用 `screenshot` 截图保存证据
- 每个测试用例执行完毕后，记录通过/失败状态

### TC-AUTH-009：Token 过期自动登出

| 项目 | 内容 |
|------|------|
| **用例编号** | TC-AUTH-009 |
| **测试场景** | 验证 token 过期后自动跳转登录页 |
| **优先级** | P2 |
| **前置条件** | 已登录 |
| **测试步骤** | 1. 通过 eval 读取 `localStorage.getItem('legacy-graph-user')`，获取其中 `accessToken`（JWT）<br>2. 构造一个过期的 JWT（修改 payload 的 exp 为过去时间，签名可不变），写回 localStorage<br>3. 刷新页面或点击任意菜单触发路由守卫 |
| **预期结果** | 1. 路由守卫 `isTokenExpired()` 检测到过期<br>2. 自动清除 localStorage 中 `legacy-graph-user` 登录态<br>3. 跳转到 /login 页面 |
| **验证方式** | 重定向到登录页 + localStorage 已清空 |
