## A. 性能（按耗时从大到小）

### 1. ⭐ `runTestGenerate` 串行 → 并发（1044s → 预计 ~260-350s）
- 位置：`AiScanOrchestrator.java:974`，纯 for 循环，最多 60 节点（20 ApiEndpoint + 20 Feature + 20 Controller）逐个阻塞调 LLM，**未用任何 executor**。
- 306 条/1044s ≈ 每节点 17s 串行。
- 此时 doc/code/mapping 都已结束，`docExtractExecutor`(4 路) 空闲，可直接复用并发。
- 风险：注释 `:301` 提到 DeepSeek 8 路并发触发限流。建议并发度 **3-4**（复用 4 路 fixed pool 即可，不要回到 8 路），LLM 客户端层应有重试退避。
- 预期：60 节点 / 4 路 ≈ 15 轮 × 17s ≈ 255s。

### 2. `persistTestCase` 单条 insert × 306
- `:1019` 每条用例一次 `testCaseRepository.insert`。并发化后 DB 写入可与 LLM 调用重叠，压力不大；如需进一步优化可改批量 insert（需引入 IService 或 SqlSession 批处理）。

### 3. `codeExtractExecutor` 是死代码
- `:149` 创建了第二个 4 线程池，但 `runCodeExtract`(`:542`) 误用 `docExtractExecutor`，`codeExtractExecutor` 只在 `shutdown()` 出现。**创建了从不用**。要么删掉减误导，要么真正用于 doc+code 并发（需先解决限流）。

### 4. `runFeatureMapping` 每批重传全量上下文
- `:779-780` 8 批每批都把**全量** `pageSummary`+`apiSummary` 塞进 prompt，输入 token 冗余。可按批内 Feature 相关性裁剪目标集。

## B. 数据质量（影响查询正确性）

### 5. ⭐ 1743 条边 `edgeType` 属性为 null（关系标签却正确）
- 实测：1743 条 `Feature-[IMPLEMENTED_BY]->ApiEndpoint` 边，**关系标签 `type(r)=IMPLEMENTED_BY` 正确，但 `r.edgeType` 属性为 null**。另有 `FeatureModule->Feature` 185、`BusinessObject->Service/Table` 22 等同样 null，合计 1956 条。
- 根因（已定位）：`Neo4jWriteRepository.mergeEdgesSubBatch`(`:542`) 的 row map（`:546-561`）**没有 `edgeType` 字段**，`ON CREATE SET r = row` 因此永远不写 `r.edgeType` 属性，只设了动态关系标签。对比单条 `mergeEdge`(`:234`) 显式 `SET r.edgeType = $edgeType` 是对的；批量路径漏了。
- 影响：所有按 `r.edgeType` 属性过滤的查询（系统总览 `SystemOverviewIngestService.buildRelationsFromGraph`、边类型统计）**漏掉这 1956 条边**——图谱投影失真。
- 修复：(a) `mergeEdgesSubBatch` row 加 `edgeType`，ON MATCH SET 也补 `r.edgeType`；(b) 存量一次性回填 `MATCH ()-[r]->() WHERE r.edgeType IS NULL SET r.edgeType = type(r)`。

### 6. `runTestGenerate` httpMethod 硬编码 "GET"
- `:981` `req.setHttpMethod("GET")` 对所有节点（含 POST/PUT/PATCH 接口）都生成 GET 测试。ApiEndpoint 节点本身带 method 属性，应读取。测试用例质量缺陷。

### 7. 知识缺口 835 条偏噪
- 50s 不慢，但 835 条缺口审核负担重。建议提高 gap 阈值或按业务域聚合去重。

## C. 覆盖与效果

### 8. 测试生成覆盖浅
- `MAX_TEST_GEN_NODES=20`(`:123`)，Feature 581 只取前 20 个（3.4% 覆盖）。提速后（点 1）可提高上限，或按入度/重要度采样。

### 9. `GRAPHIFY_ANALYZE` 启用但工具未装
- scope 开了但 `SKIPPED`（"Graphify not available"）。要么装上用起来，要么从默认 scope 移除避免误导。

## D. 增量重扫

### 10. `cachedExtract` 缓存已就绪，建议加命中率监控
- `:1291` 已实现按内容哈希的 Redis 缓存（7 天 TTL、空结果不缓存、降级直调）。建议加 cache hit/miss 指标，确认重扫未变内容时确实命中。
