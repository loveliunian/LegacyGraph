## 🚀 部署指南

### 环境要求

| 软件 | 版本要求 | 说明 |
|------|----------|------|
| JDK | 21+ | Java 运行环境 |
| Maven | 3.8+ | 构建工具 |
| Node.js | 18+ | 前端构建 |
| PostgreSQL | 16+ | 主数据库 |
| pgvector | - | PostgreSQL 向量扩展 |
| Neo4j | 5.x | 图数据库 |
| Redis | 6.x+ | 缓存 |
| MinIO | RELEASE.2023-* | 对象存储 |
| Docker | 20.x+ | 容器化（可选） |
| Docker Compose | 2.x+ | 容器编排 |

### 一键部署（仅前后端应用）

> `deploy/docker-compose.yml` **只构建前后端应用**；PostgreSQL、Neo4j、Redis、MinIO 使用**外部服务器**，连接信息经 `.env` 注入。

```bash
cd deploy
cp .env .env   # 填入外部 PG/Neo4j/Redis/MinIO 连接信息
docker compose up --build -d
```

这会构建并启动：
- backend（Spring Boot 应用）
- frontend（Nginx 静态服务）

外部依赖服务（PostgreSQL+pgvector、Neo4j、Redis、MinIO）需自行准备并在 `.env` 配置地址。若需临时拉起，可参考 `doc/LegacyGraph 部署文档.md` 的「独立部署各依赖服务」。

访问：
- 前端: http://localhost (FRONTEND_PORT，默认 80)
- 后端: http://localhost:8080 (BACKEND_PORT)

### 数据库初始化

> 依赖服务在外部时，连接到外部 PostgreSQL 执行初始化脚本（与 `.env` 中 `POSTGRES_URL` 一致）。

```bash
# 连接外部 PostgreSQL
psql -h <PG_HOST> -p 5432 -U legacy_graph -d legacy_graph

# 启用 pgvector 扩展
CREATE EXTENSION IF NOT EXISTS vector;

# 执行初始化脚本
\i /path/to/LegacyGraph/docs/sql/init.sql
```

> `init.sql` 会创建所有表（含运行时链路表 `lg_runtime_trace`），并插入初始数据（admin 用户、默认角色、默认配置）

### 后端构建和启动

```bash
cd backend
mvn clean package -DskipTests

# 构建成功后 target/legacygraph-api-1.0.0-SNAPSHOT.jar
java -jar target/legacygraph-api-1.0.0-SNAPSHOT.jar \
  --spring.config.location=/path/to/application-prod.yml
```

### 前端构建和部署

```bash
cd frontend
npm install
npm run build

# 把 dist 目录部署到 Nginx
cp -r dist/* /usr/share/nginx/html/legacygraph/
```

### Nginx 配置示例

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /usr/share/nginx/html/legacygraph;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 反向代理
    location /lg/ {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # WebSocket
    location /ws/ {
        proxy_pass http://localhost:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

### 配置文件

后端配置文件位置：
`backend/src/main/resources/application.yml`

关键配置项：

```yaml
# 数据库连接
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/legacy_graph
    username: postgres
    password: your-password

# OpenAI API (LLM 必需)
spring:
  ai:
    openai:
      api-key: your-openai-api-key
      base-url: https://api.openai.com/v1
      chat:
        options:
          model: gpt-4o
          temperature: 0.7

# Neo4j
spring:
  neo4j:
    uri: bolt://localhost:7687
    authentication:
      username: neo4j
      password: your-neo4j-password

# MinIO
minio:
  endpoint: http://localhost:9000
  access-key: minioadmin
  secret-key: minioadmin
  bucket-name: legacygraph

# Redis
spring:
  data:
    redis:
      host: localhost
      port: 6379
      password: your-redis-password
```

## 📊 监控

### 日志位置

默认日志输出到 stdout，可以通过 Docker 查看：

```bash
# 查看后端日志
docker logs -f legacygraph-backend

# 查看 Nginx 日志
docker logs -f legacygraph-nginx
```

如果直接运行 Jar 包，可以重定向日志到文件：

```bash
java -jar app.jar > app.log 2>&1 &
```

### 关键日志关键词

排查问题时关注：

- `ERROR` - 错误信息
- `WARN` - 警告，可能预示问题
- `slow request` - 慢请求，需要优化
- `Exception` - 异常堆栈

### 健康检查

后端暴露 Spring Boot Actuator 健康检查：

```
GET /actuator/health
```

返回：
```json
{
  "status": "UP",
  "components": {
    "db": { "status": "UP" },
    "neo4j": { "status": "UP" },
    "redis": { "status": "UP" }
  }
}
```

### 常用监控命令

```bash
# 查看端口监听
netstat -tlnp | grep -E '(java|nginx)'

# 查看磁盘使用
df -h

# 查看内存使用
free -h

# 查看进程
ps aux | grep java
```

## 🔍 故障排查

### 1. 后端启动失败

**症状：** 启动后立即退出

**排查步骤：**
```bash
# 查看错误日志
tail -n 100 app.log

# 常见原因：
# 1. 端口被占用
lsof -i :8080
# 杀掉占用进程
kill -9 <PID>

# 2. 数据库连接失败
# 检查数据库是否运行
pg_isready -h localhost -p 5432
# 检查用户名密码是否正确

# 3. 检查 JDK 版本
java -version
# 需要 21+
```

### 2. LLM 调用失败

**症状：** Agent 执行失败，返回空或错误

**排查：**
1. 检查 OpenAI API Key 是否正确配置
2. 检查网络是否能访问 api.openai.com
3. 检查账户余额是否充足
4. 查看日志中错误信息：
   - 401: API Key 错误
   - 429: 频率限制/配额用完
   - 500: OpenAI 服务端错误

**解决：**
```bash
# 测试 API 连通性
curl https://api.openai.com/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

### 3. 前端页面 404

**症状：** 刷新页面显示 404

**原因：** Vue Router 使用 history 模式，需要 Nginx 配置正确

**解决：**
检查 Nginx 配置是否包含：
```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

### 4. 前端 API 请求 502

**症状：** Nginx 返回 502 Bad Gateway

**原因：** 后端服务没有运行

**解决：**
```bash
# 检查后端是否运行
ps aux | grep java
# 检查端口是否监听
netstat -tlnp | grep 8080
# 重启后端服务
```

### 5. Neo4j 连接失败

**症状：** 启动时报错 "Unable to connect to Neo4j"

**排查：**
```bash
# 检查 Neo4j 是否运行
docker ps | grep neo4j
# 检查端口
nc -zv localhost 7687
# 检查用户名密码
```

### 6. pgvector 找不到

**症状：** 启动时报错 "extension vector does not exist"

**解决：**
```sql
-- 进入数据库，创建扩展
CREATE EXTENSION IF NOT EXISTS vector;
```

如果创建失败，说明 PostgreSQL 没有安装 pgvector 扩展，需要：
```bash
# Debian/Ubuntu
apt install postgresql-16-pgvector

# RHEL/CentOS
dnf install pgvector_16
```

### 7. 审计日志不显示数据

**症状：** `/lg/audit/list` 返回空

**原因：** 数据库表还没创建

**解决：**
```sql
-- 执行 sys_operation_log 表创建语句
-- 在 docs/sql/init.sql 末尾
```

### 8. 系统管理页面 API 404

**症状：** 访问 `/lg/system/users` 返回 404

**原因：** 代码没有重新编译部署

**解决：**
```bash
cd backend
mvn clean package
# 重新部署 Jar
```

## 🗂️ 备份恢复

### 数据库备份

```bash
# 每天定时备份
pg_dump -U postgres legacy_graph > backup/legacy_graph_$(date +%Y%m%d).sql

# 压缩
gzip backup/legacy_graph_$(date +%Y%m%d).sql
```

### Neo4j 备份

```bash
# 使用 Neo4j 自带工具
neo4j-admin backup --backup-dir=/backup --database=neo4j
```

### 恢复

```bash
# 恢复 PostgreSQL
psql -U postgres legacy_graph < backup/xxxx.sql

# 解压压缩包
gunzip backup/xxxx.sql.gz
```

### MinIO 备份

MinIO 数据存储在容器卷，定期备份整个卷即可：

```bash
docker run --rm --volumes-from deploy-minio-1 \
  -v $(pwd)/backup:/backup alpine \
  tar -czf /backup/minio-data-$(date +%Y%m%d).tar.gz /data
```

## 🔐 安全建议

1. **修改默认密码**
   - PostgreSQL 密码
   - Neo4j 密码
   - Redis 密码
   - MinIO 密码
   - 数据库 admin 用户密码

2. **限制网络访问**
   - 数据库端口不直接暴露公网
   - 只允许应用服务器访问

3. **启用 HTTPS**
   - 使用 Let's Encrypt 免费证书
   - 强制 HTTPS 访问

4. **定期更新**
   - 及时更新 JDK 安全补丁
   - 更新 PostgreSQL 等依赖软件

5. **API Key 安全**
   - OpenAI API Key 不要提交到代码仓库
   - 使用环境变量或配置中心管理

## 📈 性能优化

### 数据库优化

1. **索引**：所有外键都已经创建索引，保持即可
2. **VACUUM**：PostgreSQL 定期自动 VACUUM，不需要特别操作
3. **连接池**：Spring Boot 默认连接池，根据并发调整大小：
```yaml
spring:
  datasource:
    hikari:
      maximum-pool-size: 20
```

### 后端优化

1. **JVM 参数**：
```bash
java -Xms1g -Xmx2g -jar app.jar
```

2. **异步任务**：已经配置三个专用线程池，适应不同类型任务

### 前端优化

1. **启用 gzip** 在 Nginx 配置：
```nginx
gzip on;
gzip_types text/css application/javascript application/json;
```

2. **静态资源 CDN 加速**

## 📝 日常维护操作

### 添加新用户

1. 在前端系统管理页面创建，或者 SQL：
```sql
INSERT INTO sys_user (id, username, password, nickname, email, status)
```

```sql
TRUNCATE sys_operation_log;
```

或者通过 API：
```
DELETE /lg/audit/clear
```

### 查看当前连接数

```sql
SELECT count(*) FROM pg_stat_activity;
```

## 📋 常见问题 FAQ

**Q: 默认 admin 密码是什么？**
A: `admin123`，首次登录后请修改密码。

**Q: 需要多少磁盘空间？**
A: 取决于项目数量和代码量，建议至少预留 50GB。

**Q: 支持多个项目吗？**
A: 支持，LegacyGraph 原生支持多项目。

**Q: 需要多少内存？**
A: 建议分配至少 4GB 给 JVM，加上 Neo4j 等，总内存建议 8GB+。

**Q: 可以离线使用吗？**
A: 需要连接 OpenAI API 进行 LLM 调用，如果使用本地 LLM 需要自行修改代码集成。
