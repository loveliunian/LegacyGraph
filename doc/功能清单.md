# 业务图谱管理 — 功能清单

## 一、功能总览

| 一级功能 | 二级功能 | 三级功能 | 功能描述 |
|---------|---------|---------|---------|
| 业务图谱管理 | QA问答 | 流式智能问答 | 基于知识图谱与大模型的流式问答系统，通过 SSE 实时推送回答内容，回答自动附带证据链溯源；支持多轮对话上下文管理，问答历史可追溯 |
| | | 对话历史管理 | 按项目维护多轮对话列表，支持查看对话消息详情、删除历史对话，对话状态持久化存储 |
| | | 答案反馈评价 | 用户可对每条问答结果提交"有用/无用"反馈，支持反馈文本描述、标记使用的证据 ID，反馈数据用于持续优化问答质量 |
| | | 代码理解报告 | 通过 LLM Agent 对指定代码范围生成结构化理解报告，采用四阶段查询计划（结构化查询→深度阅读→功能查询→摘要生成），支持异步创建、状态查询和 Markdown 下载 |
| | | 向量语义检索 | 对文档/代码片段进行向量化存储，支持语义相似度检索和相似节点发现，辅助去重与关联分析 |
| | | LLM 提供商管理 | 配置和管理多个大模型提供商（如 OpenAI、通义千问等），支持切换默认模型、API Key 管理、模型参数调优 |
| | | 提示词模板管理 | 对 LLM 提示词模板进行增删改查、版本管理与激活/停用，支持按场景分类配置，模板变更即时生效 |
| | | Agent 执行与历史 | 通用 Agent 运行入口，按类型动态路由到对应执行器（图谱合并、测试生成等），支持运行历史查询与结果追踪 |
|  | 接入与扫描 | 代码仓库接入 | 接入 Git 代码仓库，配置仓库地址、分支、认证信息，支持连接测试与 CRUD 管理，扫描时自动拉取最新代码 |
| | | 数据库接入 | 接入目标数据库（PostgreSQL/MySQL/MariaDB），配置连接信息，自动导入 Schema 元数据（表、列、索引、外键），支持表过滤（include/exclude 通配符），自动计算 Schema 指纹（SHA-256）检测变更 |
| | | 文档资料接入 | 上传项目相关文档资料，自动切分为文档片段并支持 CRUD 管理，为图谱扫描提供非代码类知识源，支持文档类型分类与状态跟踪 |
| | | 扫描版本管理 | 创建和管理扫描版本，查看各版本的扫描进度、状态与历史记录，支持全量扫描的启动/暂停/取消/恢复操作，扫描完成后自动触发代码理解增强 |
| | | 密钥脱敏扫描 | 扫描过程中自动检测源码/配置/连接串中的敏感信息（私钥、AWS Key、GitHub Token、API Key、JDBC URL、密码赋值），按隐私级别（SECRET/CONFIDENTIAL/INTERNAL）自动脱敏后入库，防止图谱成为高敏资产聚合点 |
| | | 性能报告 | 生成扫描性能报告，包含节点数量、关系数量、扫描耗时、各阶段耗时占比，支持历史版本对比分析 |
| | | Graphify 导入 | 支持通过 Graphify 格式导入外部图谱数据，解析 JSON 格式的节点和边，与现有图谱合并 |
| | | 运行链路采集 | 基于运行时 Trace 数据采集服务间调用拓扑，记录实际链路数据，为图谱提供运行时证据 |
| | | 项目概览 | 展示项目级统计信息，包括节点总数、关系总数、证据数量、审核状态分布、最近扫描状态 |
|  | 图谱管理 | 统一图谱 | 以统一视图展示项目的完整知识图谱，包含 10 种节点类型（业务域、业务流程、功能模块、功能点、API 接口、Controller、Service、Mapper、SQL 语句、数据库表），支持按节点类型、审核状态、置信度阈值过滤，点击节点可查看证据链详情 |
| | | 业务图谱视图 | 按业务域维度展示知识图谱，输出业务节点（业务模块、业务流程、业务规则）及其关联关系，支持业务功能到代码实现的映射追踪，业务对象到数据库表的映射分析 |
| | | 功能图谱 | 按功能模块维度展示知识图谱，呈现模块下的功能节点及其调用与依赖关系，支持功能切片的可视化展示 |
| | | 代码图谱 | 以代码结构维度展示知识图谱，呈现类、方法、接口之间的调用链路，支持从 Controller 到 Service 到 Mapper 的全链路追踪 |
| | | 数据血缘 | 展示数据库表的上下游依赖关系，支持三种视图模式（完整血缘、仅上游、仅下游），提供表级影响分析（关联 API 数量、Service 数量、上下游表数量），自动识别高风险表并给出变更建议 |
| | | 运行链路 | 基于运行时 Trace 数据展示服务间调用拓扑与实际链路，支持链路详情查看与性能指标分析 |
| | | API 调用链 | 从指定 API 端点出发，追踪完整的调用链路（API→Controller→Service→Mapper→SQL→Table），支持正向追踪与反向追溯 |
| | | 表影响分析 | 分析指定数据库表变更的影响范围，输出受影响的 API、Service、Mapper、SQL 语句列表，提供变更风险评估与处置建议 |
| | | 版本差异对比 | 对比两个扫描版本的节点和边差异，高亮新增/删除/修改的节点与关系，支持差异详情查看与导出 |
| | | 节点去重 | 基于语义相似度检测重复节点，提供合并建议与一键合并功能，维护图谱数据一致性 |
| | | 知识断言管理 | 管理图谱中的知识断言（KnowledgeClaim），按证据充分性标记状态（已证实/待验证/已驳回），支持断言的版本追溯与冲突检测 |
| | | 知识缺口发现 | 自动识别图谱中的知识缺口（GapTask），按优先级排序，支持手动创建缺口任务并分配处理 |
|  | 证据与审核 | 证据工作台 | 集中管理图谱节点的证据链，支持证据查看、上传、标记失效与冲突处理，证据类型包括源码片段、文档引用、运行时 Trace、人工标注 |
| | | 事实列表 | 查看和管理知识图谱中的事实断言，按类型、状态、来源进行筛选，支持事实的创建、编辑、删除与状态变更 |
| | | 证据检索 | 搜索和检索图谱节点的关联证据，支持多条件组合查询（节点类型、证据类型、置信度范围、时间范围），支持全文检索与语义检索 |
| | | 审核队列 | 管理员创建审核任务并分配审核员，审核员对节点执行审批、驳回或评论，支持批量审核与审核意见模板 |
| | | 审核历史 | 查看已完成的审核记录，追溯审核决策过程与结果，支持按审核员、时间范围、审核结果筛选 |
| | | 冲突检测 | 自动检测图谱中的证据冲突（如同一节点的多条证据相互矛盾），提供冲突详情与解决建议 |
| | | 变更任务中心 | 管理图谱变更任务（如节点合并、关系修正、证据补充），支持任务创建、分配、状态跟踪与结果验证 |
| | | 审计日志 | 记录系统全量操作日志，包括用户登录、数据变更、审核操作、扫描任务等，支持按用户、操作类型、时间范围查询与统计 |
| | | 通知中心 | 通过 SSE 实时推送系统通知（扫描完成、审核任务分配、冲突检测告警），支持通知标记已读与历史记录查看 |
|  | 测试与报告 | 测试用例 | 管理测试用例的创建、编辑、查询与批量操作，支持 AI 自动生成测试用例（基于代码理解与图谱分析），测试用例关联图谱节点与证据 |
| | | 测试执行 | 运行测试用例并记录执行结果（通过/失败/跳过），查看执行详情（断言结果、耗时、日志输出）与历史运行记录，支持测试覆盖率统计 |
| | | 验证报告 | 查看图谱数据校验结果，包括节点完整性、关系一致性、证据充分性等规则的检查报告，支持规则自定义与阈值配置 |
| | | 迁移风险 | 识别和评估系统迁移过程中的潜在风险（如 API 变更、表结构变更、依赖缺失），提供风险等级（高/中/低）与处置建议，支持风险追踪与闭环管理 |
| | | 操作日志 | 记录系统全量操作日志，支持按用户、操作类型、时间等条件查询与统计，日志数据可导出 |
| | | 插件扩展 | 支持通过插件机制扩展测试与报告功能，插件可注册自定义校验规则、报告模板、数据导出格式 |

## 三、关键技术特性

### 3.1 数据安全
- **密钥脱敏**：扫描阶段自动检测并脱敏敏感信息，支持 7 种密钥类型识别
- **隐私分级**：SECRET / CONFIDENTIAL / INTERNAL 三级隐私保护
- **审计追溯**：全量操作日志记录，支持合规审计

### 3.2 智能化能力
- **多阶段查询计划**：结构化查询 → 深度阅读 → 功能查询 → 摘要生成
- **代码理解增强**：扫描后自动对高复杂度/高入度节点执行深度分析
- **知识缺口发现**：自动识别证据不足的节点并生成补全任务

### 3.3 图谱质量保障
- **置信度阈值**：节点与关系携带置信度评分，支持按阈值过滤展示
- **审核流程**：待审核 → 已批准 / 已驳回，支持批量审核
- **冲突检测**：自动识别证据冲突并提供解决建议

### 3.4 变更影响分析
- **表影响分析**：从表变更出发，追踪 API→Service→Mapper→SQL 全链路影响
- **API 调用链**：从 API 端点出发，追踪完整调用链路
- **版本差异对比**：高亮显示版本间的节点与关系变更

### 3.5 运行时融合
- **Trace 数据采集**：接入运行时调用链路，为图谱提供动态证据
- **运行链路可视化**：展示实际服务调用拓扑，补充静态分析盲区
