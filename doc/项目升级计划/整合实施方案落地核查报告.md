**偏移详情 · S2-T1**：
- 现象：`JavaMemberCallResolver.java` 没有 lambda / 方法引用的专门处理逻辑（grep 无 `LambdaExpr` / `MethodReferenceExpr` / `methodRef` / `visitLambda`）
- 影响：原先 CALLS 边漏的 30~70% 主要是 lambda 内部的方法调用和方法引用链（`this::foo`、`(x) -> service.foo(x)`），这一类场景仍会漏边
- 建议：在 `JavaMemberCallResolver` 增加对 `LambdaExpr` / `MethodReferenceExpr` 的 `visit()` 重写，把 lambda body 内的方法调用和方法引用解析为调用边
- 优先级：建议作为 P1 补丁尽快补上

**结论**：Sprint 2 5/6 完成，存在 1 处真实偏移（S2-T1）。

## 8. 风险与建议

### 8.1 偏移修复（建议立即处理）

**P1 · S2-T1 lambda/方法引用增强**

建议在 `JavaMemberCallResolver` 中增加：
```java
@Override
public void visit(LambdaExpr n, Void arg) {
    n.getBody().ifExpr(e -> e.accept(this, arg));
    n.getBody().ifBlockStmt(b -> b.getStatements().forEach(s -> s.accept(this, arg)));
    super.visit(n, arg);
}

@Override
public void visit(MethodReferenceExpr n, Void arg) {
    // 处理 this::foo / Service::method 等
    super.visit(n, arg);
}
```

并新增 `JavaMemberCallResolverTest` 测试用例覆盖：
- `() -> service.foo()` 的方法调用解析
- `this::bar` 的方法引用解析
- lambda 捕获变量追踪（外部变量为参数的隐式边）

### 8.2 测试补齐（建议 S5 内处理）

按上表优先级排序补齐单元测试，特别注意：
- **ScanLockService 并发测试**：mock 第二个并发调用应被拒绝
- **DbPasswordBackfillRunner 幂等测试**：第二次执行不应重复回填
- **RetrievalIntentRouter 强制引用测试**：答案未引用节点 ID 应触发重试/降级

### 8.3 文档收紧

- 综合报告与整合实施方案里的"已落地"措辞，建议改为"24 任务中 23 项落地 + 1 项偏移（S2-T1）"
