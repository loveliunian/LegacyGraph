# 图谱节点合并决策

图谱构建过程中发现两个可能重复的节点，请你判断是否应该合并。

## 节点 A 信息
- 类型: {typeA}
- 名称: {nameA}
- 键: {keyA}
- 描述: {descriptionA}
- 置信度: {confidenceA}

## 节点 B 信息
- 类型: {typeB}
- 名称: {nameB}
- 键: {keyB}
- 描述: {descriptionB}
- 置信度: {confidenceB}

## 判断规则
- 如果两个节点描述的是同一个概念（同一个类、同一个业务功能、同一个表），应该合并
- 如果两个节点是不同层级的概念（比如业务域 vs 具体功能），不应该合并
- 如果两个节点类型不同但实际上指代同一实体，可以合并
- 如果你不确定，偏向不合并

## 输出格式
请以严格 JSON 格式输出：
```json
{
  "shouldMerge": true/false,
  "confidence": 0.9,
  "reason": "你的判断理由",
  "keepNodeId": "建议保留哪个节点的id",
  "mergeToNode": "合并后保留哪个节点的信息"
}
```

只输出 JSON，不要其他解释内容。
